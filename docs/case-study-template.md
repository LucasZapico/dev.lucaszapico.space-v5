# Case Study / Project Work Template

A skeleton for documenting portfolio projects and case studies.

---

## Required Sections

### 1. Overview
- **Project title**
- **Client/Company** (if applicable)
- **Timeline** (duration, year)
- **Role** (your specific role)
- **Team size** (solo, small team, large team)
- **One-liner** - Single sentence summary

### 2. Problem Statement
- What challenge or pain point existed?
- Who was affected?
- What were the business/user consequences of not solving it?
- Any constraints or limitations?

### 3. Goals & Objectives
- Primary goal
- Secondary goals
- Success metrics (KPIs)
- Stakeholder expectations

### 4. Process / Approach
- Research phase (user interviews, competitive analysis, data review)
- Discovery findings
- Strategy/planning decisions
- Why this approach over alternatives?

### 5. Solution
- High-level solution description
- Key features/components
- Technical decisions and rationale
- Architecture overview (if relevant)

### 6. Implementation
- Tech stack used
- Notable technical challenges
- Iterations and pivots
- Collaboration highlights

### 7. Results & Impact
- Quantitative results (metrics, percentages, numbers)
- Qualitative results (feedback, testimonials)
- Before/after comparison
- Business impact

### 8. Learnings & Reflections
- What went well?
- What would you do differently?
- Skills developed
- Key takeaways

## Writing Tips

1. **Lead with impact** - Start with the most impressive result
2. **Be specific** - "Increased conversion by 34%" > "Improved conversion"
3. **Show your thinking** - Document why, not just what
4. **Keep it scannable** - Use headers, bullets, and visuals
5. **Tell a story** - Problem → Process → Solution → Result
6. **Be honest** - Include challenges and what you learned

## Example Structure for Different Project Types

### Development Project
1. Overview → 2. Challenge → 3. Technical Approach → 4. Architecture → 5. Implementation → 6. Performance/Results → 7. Learnings

### Design Project
1. Overview → 2. Research → 3. Ideation → 4. Design System → 5. Final Designs → 6. User Testing → 7. Results

### Consulting/Strategy
1. Overview → 2. Situation Analysis → 3. Recommendations → 4. Implementation Support → 5. Outcomes
